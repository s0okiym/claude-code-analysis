# 子领域分析：与原始代码对比

## 1. 功能概述
对比 Python 移植版本与原始 TypeScript 版本。

## 2. 规模对比
| 指标 | 原始 (TS) | 移植 (Py) |
|------|-----------|-----------|
| 文件数 | ~1900 | 8 |
| 代码行数 | ~512K | ~200 |
| 功能 | 完整 | 基础 |
| 复杂度 | 高 | 低 |

## 3. 架构对比
### 3.1 原始架构
```
main.tsx
  ├── REPL (交互)
  ├── QueryEngine (SDK)
  ├── Bridge (远程)
  └── query.ts (核心循环)
        ├── claude.ts (API)
        ├── tools/ (30+ 工具)
        ├── compact/ (压缩)
        ├── permissions/ (权限)
        └── mcp/ (MCP)
```
### 3.2 移植架构
```
main.py
  ├── port_manifest.py
  ├── query_engine.py
  ├── commands.py
  └── tools.py
```

## 8. 结论
移植版本是原始代码的极简实现，保留了核心概念但移除了复杂功能。适合：
- 学习核心架构
- 快速原型
- 渐进开发

不适合：
- 生产使用
- 完整功能
- 复杂场景
