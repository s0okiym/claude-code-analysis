## 4. 关键设计
### 4.1 为什么渐进？
| 方式 | 优点 | 缺点 |
|------|------|------|
| 渐进 | 可交付 | 周期长 |
| 大爆炸 | 完整 | 风险高 |
| 原型 | 快速验证 | 需重写 |

**选择渐进**:
- 持续价值
- 风险可控
- 反馈驱动
### 4.2 扩展原则
1. **向后兼容**: 不破坏现有 API
2. **可选**: 新功能可开关
3. **文档**: 更新文档
4. **测试**: 覆盖新功能

```python
# 扩展点设计
class ExtensibleEngine:
    """可扩展引擎"""
    
    def __init__(self):
        self.plugins = []
    
    def register(self, plugin):
        """注册插件"""
        self.plugins.append(plugin)
    
    def execute(self, task):
        """执行，调用插件"""
        for plugin in self.plugins:
            plugin.before_execute(task)
        
        result = self._execute(task)
        
        for plugin in reversed(self.plugins):
            plugin.after_execute(result)
        
        return result
```
